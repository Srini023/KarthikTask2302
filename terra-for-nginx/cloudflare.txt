To connect your Cloudflare domain to your Nginx deployment, you need to point your domain's DNS records to the external IP address created by your Kubernetes Ingress controller.

You can manage this manually via the Cloudflare dashboard, or fully automate it by adding the Cloudflare Terraform Provider directly into your configuration.

Method 1: The Automated Way (Recommended)

You can manage your Cloudflare DNS records directly inside your Terraform project by updating your files with the configurations below.

1. Add the Cloudflare Provider to providers.tfAdd the Cloudflare block inside your required_providers and configure the provider using a Cloudflare API token.

terraform {
  required_providers {
    # ... keep your existing docker, google, kubernetes, and helm providers ...
    cloudflare = {
      source  = "cloudflare/cloudflare"
      version = "~> 4.0"
    }
  }
}

variable "cloudflare_api_token" {
  type        = string
  sensitive   = true
  description = "Your Cloudflare API Token with Zone.DNS permissions"
}

variable "cloudflare_zone_id" {
  type        = string
  description = "The Zone ID for your domain found on the Cloudflare dashboard overview page"
}

provider "cloudflare" {
  api_token = var.cloudflare_api_token
}

2. Create the DNS Record in main.tfAppend this block to the bottom of your main.tf file. It automatically reads the public IP generated by your Ingress LoadBalancer and creates the record in Cloudflare.
resource "cloudflare_record" "nginx_dns" {
  zone_id = var.cloudflare_zone_id
  name    = "nginx"                     # Creates ://yourdomain.com
  value   = data.kubernetes_service.ingress_lb.status.load_balancer.ingress.ip
  type    = "A"
  proxied = false                      # MUST be false initially for Let's Encrypt HTTP-01 to validate
}


3. Update terraform.tfvarsAdd your specific Cloudflare zone details:

cloudflare_zone_id = "your_actual_cloudflare_zone_id_string"
domain_name        = "://yourdomain.com"

4. Deploy Securely via Terminal

Just like you did with your Docker password, pass your Cloudflare API token via your terminal environment to keep it secure:

bashexport TF_VAR_cloudflare_api_token="your_cloudflare_dns_api_token"
terraform apply

Method 2: The Manual Way (Without Changing Code)
If you prefer not to manage Cloudflare through code, you can easily map it manually:
Run terraform apply to deploy your setup.Copy the IP address printed out by the terminal variable ingress_public_ip.
Log into your Cloudflare Dashboard and select your domain name.
Go to DNS > Records > click Add Record.
Set the fields exactly like this:
Type: A
Name: nginx (or @ if you want the root domain)
IPv4 address: Paste your copied ingress_public_ip
Proxy status: DNS Only (Turn the orange cloud OFF). Let's Encrypt needs this turned off initially to verify your domain ownership.
Click Save.

⚠️ Critical Cloudflare & Cert-Manager Rule

Because your Terraform code uses Cert-Manager with an http01 challenge, Cloudflare's proxy features can break the certificate issuance process.

Always ensure Proxy Status is turned OFF (Grey Cloud) when deploying for the first time.

Once Cert-Manager successfully completes the challenge and generates your SSL padlock (this takes 2–5 minutes), you can change proxied = true (or flip the orange cloud on manually) to take advantage of Cloudflare's DDoS protection and CDN caching.

Inside your Cloudflare dashboard under SSL/TLS, ensure your encryption mode is set to Full (strict) to avoid infinite redirect loops.
