To pass your sensitive Docker Hub password via environment variables securely, you leverage Terraform’s built-in behavior: any environment variable prefixed with TF_VAR_ is automatically read as a variable input by Terraform.This completely eliminates the need to write your password into terraform.tfvars or any other raw text file.

Step1: Set the Environment Variable Before running any Terraform commands, set the environment variable in your terminal session. Make sure the name matches your variable definition exactly, but with TF_VAR_ added to the front.
On Linux / macOS:bashexport TF_VAR_docker_hub_password="your_actual_secret_docker_hub_token_or_password"
On Windows (PowerShell):powershell$env:TF_VAR_docker_hub_password="your_actual_secret_docker_hub_token_or_password"

Step2: Update your terraform.tfvarsEnsure that docker_hub_password is not written inside your terraform.tfvars file. Your terraform.tfvars file should look like this:hclgcp_project_id      = "srinisre023"
gcp_region          = "us-central1"
docker_hub_username = "2823"
domain_name         = "://yourdomain.com"
ssl_email           = "your-email@example.com"
# Leave docker_hub_password completely out of this file!

Step 3: Run Terraform NormallyBecause you have set TF_VAR_docker_hub_password in your active terminal environment, Terraform will silently pick it up behind the scenes. You do not need to pass extra flags to your commands anymore.
Simply run:bashterraform plan
terraform apply

Verification Checklist

To ensure your password remains safe and undetected:
Check the variable definition: In your variables.tf, confirm that sensitive = true is set on the docker_hub_password block. This prevents Terraform from printing your secret password to your screen or terminal logs.
Inspect your .gitignore: Make sure your .bash_history or command line tracking files don't accidentally leak secrets. If you want to keep the export command out of your bash history, type a single space before the export command in your terminal (on most Linux systems, this prevents the command from being recorded).
